# 📝 프로젝트 스펙 및 기능 명세서

본 문서는 통합 건강관리 애플리케이션의 세부 개발 스펙 및 모바일 화면에 구성될 기능들의 상세 요구사항을 정의합니다.

---

## 1. 개발 스펙 (Tech Stack)

*   **프론트엔드 프레임워크**: React (Vite 기반, 빌드 속도 및 반응성 최적화)
*   **스타일링**: Tailwind CSS + Shadcn/ui (Radix UI 기반의 컴포넌트 프레임워크)
*   **백엔드 서비스**: Supabase (PostgreSQL Database, 이메일/소셜 인증 등 지원)
*   **AI 프레임워크**: Google AI SDK (Gemini 2.5 Flash / Gemini 1.5 Flash 무료 등급 사용)
*   **호스팅 & 배포**: Cloudflare Pages (무료 정적 웹 호스팅 및 환경 변수 제공)

---

## 2. 화면 구성 및 기능 상세

이 앱은 세로 모드 모바일 웹 뷰를 중심으로 디자인됩니다. 대시보드 메인 화면에서 모든 입력을 유기적으로 유도합니다.

### 1) 상단 날짜 및 기본 정보 영역
*   **날짜 선택기 (Date Picker)**
    *   화면 가장 상단에 위치하며, 이전/다음 날짜 이동 및 달력을 통한 직접 선택 지원.
    *   날짜가 바뀌면 해당 날짜에 기록된 데이터베이스의 로그들을 실시간으로 패치하여 표시.
*   **기본 정보 (프로필 고정 영역)**
    *   생년월일(1992. 08. 19), 성별(남자), 키(167 cm) 정보를 표시.
    *   최초 가입 또는 필요시 설정 모달을 통해 변경 가능하며, DB `profiles` 테이블에 반영됨.
*   **몸무게 입력 필드 (데일리)**
    *   매일 아침 측정한 몸무게를 간편하게 입력하고 저장할 수 있는 입력란 제공.

### 2) 입력 정보 영역 (8대 데일리 입력 항목)
대시보드에 배치된 각 버튼을 클릭하면 화면 하단에서 올라오는 **바텀 시트(Bottom Sheet)**를 통해 입력을 진행합니다.

1.  **유산소 운동**
    *   입력 정보: 운동 종류(달리기, 자전거 등), 거리(km), 시간(분/초).
2.  **무산소 운동 (번핏 UI 스타일)**
    *   입력 정보: 운동 이름(예: 스쿼트, 벤치프레스).
    *   동적 세트 관리: 세트 추가 버튼을 통해 각 세트별로 무게(kg)와 횟수(reps)를 다르게 입력 가능.
3.  **스트레칭**
    *   입력 정보: 스트레칭 부위/이름(목, 어깨, 전신 등), 수행 시간(분).
4.  **음식 (식단)**
    *   입력 정보: 식사 타임 구분(아침, 점심, 저녁, 간식), 자유로운 키워드 태그 입력 (예: `["닭가슴살", "현미밥", "아보카도"]`).
5.  **영양제**
    *   입력 정보: 설정한 시간대별(아침/점심/저녁) 패키지 체크박스.
    *   작동 방식: 영양제 설정 화면에서 등록해 놓은 시간대별 리스트가 로드되며 복용 여부를 단순히 체크하여 기록.
6.  **물 (수분 섭취)**
    *   입력 정보: 수분 섭취량 (ml).
    *   작동 방식: 원터치 퀵 버튼(+250ml, +500ml 등) 제공 및 직접 텍스트 입력 가능.
7.  **수면**
    *   입력 정보: 취침 시각, 기상 시각.
    *   작동 방식: 날짜/시간 선택기를 이용해 입력하며, 총 수면 시간은 프론트엔드에서 계산하여 함께 출력.
8.  **데이터 추가 (AI 파싱 업로드)**
    *   입력 정보: 인바디 또는 건강검진 결과지 이미지/PDF 업로드.
    *   작동 방식:
        1. 이미지 업로드 시 브라우저 내에서 Base64 데이터로 변환.
        2. Gemini API로 구조화 요청을 보내 수치 텍스트만 JSON으로 추출.
        3. 파싱 결과를 사용자에게 확인 및 편집하는 중간 화면 제공.
        4. 확인 완료 후 Supabase DB에 적재하고 업로드된 파일 객체는 즉시 소멸(스토리지 비용 제로).

### 3) AI 데일리 피드백 영역
*   **"AI 피드백 받기" 버튼**
    *   해당 날짜에 입력된 유산소, 무산소, 스트레칭, 수면, 영양제, 물, 식단, 몸무게 데이터를 프롬프트로 병합.
    *   Gemini API를 호출하여 세분화된 영역별 분석 의견을 응답받아 노출.
*   **피드백 출력 구성**:
    *   **종합 피드백**: 하루의 건강 점수와 생활 습관에 대한 총평.
    *   **운동 피드백**: 유산소/무산소/스트레칭 밸런스 점검 및 조언.
    *   **영양 피드백**: 식단 밸런스, 영양제 복용 현황, 수분 섭취 부족 여부 평가.
    *   **수면 피드백**: 수면 환경 및 수면 총량의 적합성 피드백.
