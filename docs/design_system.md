# 🎨 디자인 시스템 및 UI 가이드라인

본 문서는 앱의 사용자 경험(UX)과 시각적 브랜딩(UI)의 일관성을 확보하고, 모바일 환경에서의 조작 편의성을 위한 설계 규칙을 정의합니다.

---

## 1. 서체 (Font Family)

*   **기본 폰트**: **스포카 한 산스 네오 (Spoqa Han Sans Neo)**
*   **선정 이유**: 
    - 국문과 영문, 특히 숫자가 섞여 있을 때의 자간 및 정렬도가 매우 훌륭하여 데이터 기반의 건강 대시보드에 최적입니다.
    - 웹 접근성을 향상시키며 가독성이 높습니다.
*   **적용 방법 (Vite/HTML)**:
    - HTML `<head>`에 Google Fonts 또는 Spoqa CDN 스타일시트를 주입하여 로드합니다.
    ```html
    <link href="//spoqa.github.io/spoqa-han-sans/css/SpoqaHanSansNeo.css" rel="stylesheet" type="text/css">
    ```
    - Tailwind CSS 설정(`tailwind.config.js`)의 `fontFamily`에 `'Spoqa Han Sans Neo', 'sans-serif'`를 선언하여 기본 폰트로 상속시킵니다.

## 3. 국내 모바일 UI/UX 표준 준수 사항

국내의 대형 모바일 서비스(토스, 카카오, 네이버 등) 디자인 시스템의 편의성 기준을 바탕으로 설계 규칙을 정립합니다.

### 1) 엄지손가락 영역 (Thumb Zone)과 터치 타겟
- 스마트폰 화면 하단은 한손 조작 시 엄지손가락이 가장 쉽게 닿는 영역입니다. 중요 입력 버튼(데일리 로그 입력 버튼 등)은 화면 하단 영역에 조밀하게 배치합니다.
- 오작동 및 더블 터치를 방지하기 위해, 모든 터치 가능한 요소(버튼, 아이콘 탭)는 **최소 44px × 44px 이상의 터치 타겟 크기**를 유지하고 최소 8px 이상의 간격을 둡니다.

### 2) 모달 다이얼로그 대신 바텀 시트 (Bottom Sheet) 활용
- 화면 중앙에 뜨는 모달 다이얼로그는 모바일 세로 화면에서 닫기 버튼이 손에 닿지 않아 사용자 피로도를 높입니다.
- 유산소, 무산소, 식사 입력 창 등 데이터를 입력하는 핵심 레이아웃은 화면 하단에서 슬라이드 업되는 **바텀 시트** 형태로 구현합니다. 

### 3) 폼 필드 접근성 가이드라인
- 모바일 가상 키보드가 올라올 때 화면 하단의 입력 폼이 가려지지 않도록 스크롤 여유 마진(Focus Scroll Margin)을 충분히 보장합니다.
- 숫자 입력 필드(`몸무게`, `물 섭취량`, `무게`, `횟수` 등)는 활성화 시 모바일에서 **숫자 전용 키보드(Numeric Keypad)**가 즉시 뜨도록 `inputmode="decimal"` 혹은 `type="number"` 속성을 완벽히 적용합니다.

### 4) 실시간 반응형 피드백
- 네트워크 요청(Gemini API 통신, Supabase 저장 등) 발생 시 버튼에 로딩 스피너를 보여주어 중복 클릭을 방지합니다.
- 데이터 입력 또는 성공적으로 데이터가 복구되면 화면 상/하단에 **토스트(Toast)** 팝업을 띄워 유저에게 동작 상태를 명확히 전송합니다.
